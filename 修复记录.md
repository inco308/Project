## 1. HTTPS筛选结果不同问题

### 问题描述
- Python实现筛选出104个HTTPS节点
- C实现筛选出55个HTTPS节点
- 两种实现结果不一致

### 错误原因
C实现中的`filter_https_nodes`函数从CSR图的边中读取`protocol`和`dst_port`来判断是否为HTTPS会话，但CSR图的每条边只保存了最后一个会话的信息，导致前面的HTTPS会话被覆盖。

### 修改前的代码
**文件**: `c_src/analysis.c`
```c
DynamicArray* filter_https_nodes(CSRGraph* graph, const char* csv_file) {
    if (!graph || graph->node_count == 0) return NULL;
    
    HashMap* https_node_map = hash_map_create(INITIAL_CAPACITY);
    if (!https_node_map) return NULL;
    
    // 从CSR图的边读取信息
    for (int u = 0; u < graph->node_count; u++) {
        int start = graph->row_ptr[u];
        int end = graph->row_ptr[u + 1];
        for (int i = start; i < end; i++) {
            EdgeData* edge = &graph->edges[i];
            if (edge->protocol == 6 && edge->dst_port == 443) {
                // 添加源IP和目标IP
                const char* u_ip = csr_graph_get_ip(graph, u);
                const char* v_ip = csr_graph_get_ip(graph, edge->target_node);
                // ...
            }
        }
    }
    // ...
}
```

### 修改后的代码
**文件**: `c_src/analysis.c`
```c
DynamicArray* filter_https_nodes(CSRGraph* graph, const char* csv_file) {
    if (!graph || graph->node_count == 0 || !csv_file) return NULL;
    
    // 直接从原始CSV文件读取所有会话
    DynamicArray* sessions = read_all_sessions(csv_file);
    if (!sessions) return NULL;
    
    HashMap* https_node_map = hash_map_create(INITIAL_CAPACITY);
    if (!https_node_map) {
        dynamic_array_destroy(sessions);
        return NULL;
    }
    
    // 遍历所有会话检查protocol=6且dst_port=443
    for (int i = 0; i < sessions->size; i++) {
        Session* session = (Session*)dynamic_array_get(sessions, i);
        if (session && session->protocol == 6 && session->dst_port == 443) {
            // 添加源IP和目标IP到HTTPS节点集合
            if (session->source_ip[0] != '\0') {
                int dummy;
                if (hash_map_get(https_node_map, session->source_ip, &dummy) != SUCCESS) {
                    hash_map_put(https_node_map, session->source_ip, 1);
                }
            }
            
            if (session->dest_ip[0] != '\0') {
                int dummy;
                if (hash_map_get(https_node_map, session->dest_ip, &dummy) != SUCCESS) {
                    hash_map_put(https_node_map, session->dest_ip, 1);
                }
            }
        }
    }
    
    dynamic_array_destroy(sessions);
    // ...
}
```

### 相关修改
- **文件**: `c_src/analysis.h` - 更新函数声明
  ```c
  DynamicArray* filter_https_nodes(CSRGraph* graph, const char* csv_file);
  ```

- **文件**: `c_src/main.c` - 添加全局变量保存CSV路径，并在调用时传递
  ```c
  static char global_csv_file[1024] = "";
  // ...
  strncpy(global_csv_file, argv[2], sizeof(global_csv_file) - 1);
  // ...
  DynamicArray* nodes = filter_https_nodes(global_graph, global_csv_file);
  ```

## 2. 边数不同问题

### 问题描述
- Python实现显示1240条边
- C实现显示1256条边（报告），实际收集到1204条唯一边
- 两种实现的边集合有显著差异

### 错误原因
`csr_graph_add_edge`函数将新边追加到edges数组的末尾，而不是插入到正确的位置，导致edges数组和row_ptr的对应关系被打乱！

CSR图要求：
- `row_ptr[u]` 表示节点u的第一条边在edges数组中的起始索引
- `row_ptr[u+1]` 表示节点u的最后一条边在edges数组中的结束索引

当新边被追加到末尾而不是插入到`row_ptr[source]`位置时，edges数组的结构被破坏，导致边的关联关系错误。

### 修改前的代码
**文件**: `c_src/csr_graph.c`
```c
static void csr_graph_add_edge(CSRGraph* graph, int source, int target) {
    if (!graph || source < 0 || source >= graph->node_count) return;
    
    // 直接追加到末尾 - 错误！
    EdgeData* new_edges = (EdgeData*)realloc(graph->edges, (graph->edge_count + 1) * sizeof(EdgeData));
    if (!new_edges) return;
    graph->edges = new_edges;
    
    EdgeData* edge = &graph->edges[graph->edge_count];
    edge->target_node = target;
    edge->total_data_size = 0;
    edge->total_duration = 0.0;
    edge->protocol = 0;
    edge->src_port = 0;
    edge->dst_port = 0;
    memset(edge->protocol_traffic, 0, sizeof(edge->protocol_traffic));
    
    for (int i = source + 1; i <= graph->node_count; i++) {
        graph->row_ptr[i]++;
    }
    
    graph->edge_count++;
}
```

### 修改后的代码
**文件**: `c_src/csr_graph.c`
```c
static void csr_graph_add_edge(CSRGraph* graph, int source, int target) {
    int i;
    int insert_pos;
    EdgeData* new_edges;
    EdgeData* edge;
    
    if (!graph || source < 0 || source >= graph->node_count) return;
    
    new_edges = (EdgeData*)realloc(graph->edges, (graph->edge_count + 1) * sizeof(EdgeData));
    if (!new_edges) return;
    graph->edges = new_edges;
    
    // 找到正确的插入位置
    insert_pos = graph->row_ptr[source];
    
    // 把insert_pos之后的所有边都向后移动一位
    for (i = graph->edge_count; i > insert_pos; i--) {
        graph->edges[i] = graph->edges[i - 1];
    }
    
    // 在insert_pos位置添加新边
    edge = &graph->edges[insert_pos];
    edge->target_node = target;
    edge->total_data_size = 0;
    edge->total_duration = 0.0;
    edge->protocol = 0;
    edge->src_port = 0;
    edge->dst_port = 0;
    memset(edge->protocol_traffic, 0, sizeof(edge->protocol_traffic));
    
    for (i = source + 1; i <= graph->node_count; i++) {
        graph->row_ptr[i]++;
    }
    
    graph->edge_count++;
}
```

### 验证结果
修复后：
- Python实现：226个节点，1240条边
- C实现：226个节点，1240条边
- 两种实现的边集合完全一致！

## 3. 流量排序结果不同问题

### 问题描述
- 当多个节点的总流量相同时，Python和C实现的排序顺序不同
- 例如，在第153位，Python显示'192.168.200.42'（流量155），而C显示'192.168.200.59'（流量155）

### 错误原因
- Python和C的排序函数在处理相同值的元素时，没有定义明确的次要排序条件
- C的`qsort`函数对于相等元素的相对顺序是不确定的
- Python的`sorted`函数是稳定的，但同样没有定义相同值元素的排序规则

### 修改前的代码

**C实现 - 文件**: `c_src/analysis.c`
```c
int compare_node_traffic(const void* a, const void* b) {
    if (!a || !b) return 0;
    NodeTraffic* na = (NodeTraffic*)a;
    NodeTraffic* nb = (NodeTraffic*)b;
    if (na->total_traffic > nb->total_traffic) return -1;
    if (na->total_traffic < nb->total_traffic) return 1;
    return 0;  // 没有次要排序条件
}
```

**Python实现 - 文件**: `network_analyzer.py`
```python
def sort_nodes_by_traffic(graph):
    nodes = []
    for node_id in graph.graph.nodes():
        ip = graph.id_to_ip[node_id]
        total = graph.get_node_total_traffic(node_id)
        nodes.append((ip, total))
    return sorted(nodes, key=lambda x: x[1], reverse=True)  # 没有次要排序条件
```

### 修改后的代码

**C实现 - 文件**: `c_src/analysis.c`
```c
int compare_node_traffic(const void* a, const void* b) {
    if (!a || !b) return 0;
    NodeTraffic* na = (NodeTraffic*)a;
    NodeTraffic* nb = (NodeTraffic*)b;
    if (na->total_traffic > nb->total_traffic) return -1;
    if (na->total_traffic < nb->total_traffic) return 1;
    return strcmp(na->ip, nb->ip);  // 流量相同时，按IP地址字典序排序
}
```

**Python实现 - 文件**: `network_analyzer.py`
```python
def sort_nodes_by_traffic(graph):
    nodes = []
    for node_id in graph.graph.nodes():
        ip = graph.id_to_ip[node_id]
        total = graph.get_node_total_traffic(node_id)
        nodes.append((ip, total))
    return sorted(nodes, key=lambda x: (-x[1], x[0]))  # 流量降序，IP升序
```

### 验证结果
修复后：
- 两种实现对于流量相同的节点，都按IP地址的字典序进行排序
- 排序结果完全一致！

## 总结
| 问题 | 原因 | 修复方案 |
|------|------|----------|
| HTTPS筛选结果不同 | CSR图边只保存最后一个会话的信息 | 直接从原始CSV文件读取所有会话筛选 |
| 边数不同 | csr_graph_add_edge将新边追加到末尾，而非正确位置 | 修复插入逻辑，在row_ptr[source]位置插入新边 |
| 流量排序结果不同 | 相同流量值的节点没有定义次要排序条件 | 流量相同时，按IP地址字典序排序 |
