# 网络流量分析器 - 项目架构说明

## 一、项目整体架构

本项目采用**C/Python混合架构**，核心分析算法使用C语言实现，GUI界面和辅助功能使用Python实现。

```
Project/
├── c_src/                    # C语言核心实现
│   ├── common.h             # 通用类型定义与宏
│   ├── dynamic_array.h/c    # 动态数组数据结构
│   ├── hash_map.h/c         # 哈希表数据结构
│   ├── csr_graph.h/c        # CSR稀疏图数据结构
│   ├── csv_reader.h/c       # CSV文件读取模块
│   ├── analysis.h/c         # 网络流量分析算法
│   ├── main.c               # 主程序入口（与Python通信）
│   └── build.bat            # C代码编译脚本
├── network_analyzer.exe     # 编译后的C程序
├── c_analyzer_gui.py        # 图形用户界面
├── graph_visualizer.py      # 图可视化模块
├── capture_utils.py         # 流量捕获模块
├── network_analyzer.py      # Python实现（参考版本）
└── requirements.txt         # Python依赖
```

## 二、C语言核心实现（c_src/）

### 2.1 文件组织与模块划分

C代码已按照**功能模块**清晰地组织到不同的`.h`头文件和`.c`源文件中。

| 模块 | 头文件 | 源文件 | 功能描述 |
|------|--------|--------|----------|
| 通用定义 | common.h | - | 类型定义、宏、错误码 |
| 动态数组 | dynamic_array.h | dynamic_array.c | 可动态扩容的数组 |
| 哈希表 | hash_map.h | hash_map.c | 字符串→整数的哈希映射 |
| CSR图 | csr_graph.h | csr_graph.c | 稀疏图存储与操作 |
| CSV读取 | csv_reader.h | csv_reader.c | CSV文件解析与图构建 |
| 分析算法 | analysis.h | analysis.c | 流量分析核心算法 |
| 主程序 | - | main.c | 命令行接口，与Python通信 |

### 2.2 各模块详细说明

#### common.h - 通用定义
- **功能**：定义项目通用的数据结构、宏和错误码
- **主要内容**：
  - `Session`：会话数据结构
  - `NodeTraffic`、`SuspiciousNode`：节点流量统计
  - 错误码定义：`SUCCESS`、`ERR_FILE_OPEN`、`ERR_MEMORY`等
  - 工具函数：`ip_to_uint32()`、`ip_in_range()`

#### dynamic_array.h/c - 动态数组
- **功能**：提供可自动扩容的数组容器
- **主要函数**：
  ```c
  DynamicArray* dynamic_array_create(size_t element_size, int initial_capacity);
  int dynamic_array_append(DynamicArray* arr, void* element);
  void* dynamic_array_get(DynamicArray* arr, int index);
  void dynamic_array_destroy(DynamicArray* arr);
  ```

#### hash_map.h/c - 哈希表
- **功能**：IP字符串到整数ID的映射（线性探测解决冲突）
- **主要函数**：
  ```c
  HashMap* hash_map_create(int initial_capacity);
  int hash_map_put(HashMap* map, const char* key, int value);
  int hash_map_get(HashMap* map, const char* key, int* out_value);
  ```

#### csr_graph.h/c - CSR图结构
- **功能**：使用CSR（Compressed Sparse Row）格式高效存储稀疏图
- **主要函数**：
  ```c
  CSRGraph* csr_graph_create(void);
  int csr_graph_add_session(CSRGraph* graph, Session* session);
  const char* csr_graph_get_ip(CSRGraph* graph, int node_id);
  ```

#### csv_reader.h/c - CSV读取
- **功能**：从CSV文件读取会话数据并构建图
- **主要函数**：
  ```c
  CSRGraph* read_csv_to_graph(const char* file_path);
  DynamicArray* read_all_sessions(const char* file_path);
  ```

#### analysis.h/c - 分析算法
- **功能**：核心网络流量分析算法
- **主要功能**：
  - 流量排序：`sort_nodes_by_traffic()`
  - HTTPS筛选：`filter_https_nodes()`
  - 可疑节点检测：`find_suspicious_nodes()`
  - 星型结构查找：`find_star_structures()`
  - 子图查找：`get_subgraph_by_ip()`、`get_all_subgraphs()`
  - 路径查找：`find_min_congestion_path()`、`find_min_hop_path()`
  - 安全规则检查：`check_security_rules()`

#### main.c - 主程序
- **功能**：命令行入口，通过JSON与Python通信
- **主要命令**：
  - `info`：获取图信息
  - `sort_traffic`：流量排序
  - `filter_https`：HTTPS筛选
  - `find_suspicious <ratio>`：可疑节点检测
  - `find_stars <min_edges>`：星型结构查找
  - `get_subgraph <ip>`：获取子图
  - `list_subgraphs`：列出所有子图
  - `find_path <src> <dst>`：路径查找
  - `check_security <addr1> <addr2> <addr3> <allowed>`：安全规则检查

### 2.3 C代码规范

1. **函数头注释**：每个公开函数都有统一的Doxygen风格注释
2. **关键步骤注释**：函数内部关键处理步骤都有注释说明
3. **模块划分清晰**：每个模块职责单一，通过头文件暴露接口
4. **错误处理**：使用统一的错误码返回机制

## 四、编译与运行

### 4.1 编译C程序

使用`c_src/build.bat`编译：
```batch
cd c_src
build.bat
```

### 4.2 运行GUI

```bash
# 方法1：直接运行Python
python c_analyzer_gui.py

# 方法2：双击批处理文件

```

## 五、数据流向

```
CSV文件 → csv_reader.c → CSRGraph → analysis.c → 分析结果
                                      ↓
                                  main.c → JSON输出
                                      ↓
                          c_analyzer_gui.py → 显示/可视化

PCAP文件 → capture_utils.py → CSV文件
实时流量 → tshark → PCAP文件 → capture_utils.py → CSV文件
```
